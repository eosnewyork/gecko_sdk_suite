Silicon Labs Flex Documentation

Release Notes / Quick Start Guides

  • Getting Started with the Silicon Labs Flex SDK for the Wireless Gecko (EFR32) Portfolio -- QSG138

    For the EFR32 portfolio, provides basic information on configuring, building, and installing applications using the Silicon Labs Flex SDK, which offers two paths to application development. One, using Silicon Labs Connect, provides a fully-featured, easily-customizable wireless networking solution. The other begins with Silicon Labs RAIL (Radio Abstraction Interface Layer), and offers a customizable radio interface layer designed to support proprietary or standards-based wireless protocols.

  • Flex SDK Release Notes

    Lists compatibility requirements and sources for all software componentsin the development environment. Discusses the latest changes to the SiliconLabs Flex SDK, including added/deleted/deprecated features/API. Reviews bugs that have been fixed since the last release, and any pending ones. Intended for project managers and software developers.

Application Development Fundamentals

  • Connect Fundamentals -- UG103-12

    Describes the features and functions of the Silicon Labs Connect stack, including its device types, network topologies, and its 'building block' development methodology using plugins.

  • RAIL Fundamentals -- UG103-13

    Describes the features and functions of Silicon Labs RAIL (Radio Abstraction Interface Layer). RAIL provides an intuitive, easily-customizable radio interface layer that is designed to support proprietary or standards-based wireless protocols.

  • Wireless Networking Fundamentals -- UG103-01

    Introduces some fundamental concepts of wireless networking. These concepts are referred to in other Fundamentals documents. If you are new to wireless networking, you should read this document first.

  • Design Choices Fundamentals -- UG103-03

    Discusses the major decisions to be made about architecting a wireless networking solution, as well as additional decisions to be made if you are designing a ZigBee solution.

  • Bootloading Fundamentals -- UG103-06

    Introduces bootloading for Silicon Labs networking devices. Discusses the Gecko Bootloader as well as legacy Ember and Bluetooth bootloaders, and describes the file formats used by each.

  • Non-Volatile Data Storage Fundamentals -- UG103-07

    Introduces non-volatile data storage using flash and the three different storage implementations offered for Silicon Labs microcontrollers and SoCs: Simulated EEPROM, PS Store, and NVM3.

  • Multiprotocol Fundamentals -- UG103-16

    Describes the four multiprotocol modes, discusses considerations when selecting protocols for multiprotocol implementations, and reviews the Radio Scheduler, a required component of a dynamic multiprotocol solution.

Flex References

  • Simplicity Commander Reference Guide -- UG162

    Describes how and when to use Simplicity Commander's Command-Line Interface. Only EFR32 is currently supported. EM3xx is not supported at this time.

  • Configuring Peripherals for 32-Bit Devices in Simplicity Studio -- AN1115

    Describes how to initialize and customize peripherals for 32-bit devices, both in the Simplicity IDE and using the standalone Hardware Configurator tool.

More About Connect

  • Silicon Labs Connect User Guide -- UG235

    Provides in-depth information for developers using the Silicon Labs Connect stack as a foundation for their application development on the Wireless Gecko (EFR32) portfolio or custom hardware. The Connect stack is delivered as part of the Silicon Labs Flex SDK. You should have already installed the Simplicity Studio development environment and the Flex SDK, and be familiar with the basics of configuring, compiling, and flashing Connect-based applications.

  • Silicon Labs Connect API Reference Guide

    Contains a comprehensive list of APIs used to interface to the Silicon Labs Connect stack.

  • Silicon Labs Gecko Bootloader User's Guide -- UG266

    Describes the high-level implementation of the Silicon Labs Gecko Bootloader for EFR32 SoCs and NCPs, and provides information on how to get started using the Gecko Bootloader with Silicon Labs wireless protocol stacks.

  • Using the Gecko Bootloader with Silicon Labs Connect -- AN1085

    Includes detailed information on using the Silicon Labs Gecko Bootloader with Connect. It supplements the general Gecko Bootloader implementation information provided in UG266: Silicon Labs Gecko Bootloader User's Guide.

  • Building Low Power Networks with the Silicon Labs Connect Stack -- AN902

    Illustrates techniques using the Silicon Labs Connect stack to reduce power consumption in a sensor network developed using the Silicon Labs Connect Stack.

  • Host/Network Co-Processor Communications Using Silicon Labs Thread and Connect -- AN844

    Describes how to set up and test UART communication between a host and Network Co-Processor (NCP) using ASHv3-UART. You should have a Raspberry Pi, USB cable (for UART communication), and a development board.

  • ASHv3 Protocol Reference -- UG115

    Describes the protocol used by the UART-based ASHv3 to carry commands and responses reliably between a host processor and a network co-processor.

  • Developing Connect vNCP Applications with Micrium OS -- AN1153

    Describes how the Silicon Labs Flex SDK provides support for running the Connect stack on top of the Micrium OS kernel.

  • Using Simulated EEPROM Version 1 and Version 2 for the EM35x and EFR32 SoC Platforms -- AN703

    Describes how to use the simulated EEPROM in Silicon Labs' EM35x and Mighty Gecko (EFR32MG) platforms, gives an overview of the internal design, and discusses design considerations. You should already be familiar with the token system in the HAL. For more information on HAL and the token system, see the online HAL API documentation.

  • Using Tokens for Non-Volatile Data Storage -- AN1154

    Describes tokens and shows how to use them for non-volatile data storage in EmberZNet PRO, Silicon Labs Flex, and Silicon Labs Thread applications.

More About RAIL

  • Silicon Labs RAIL API Reference Guide

    Contains a comprehensive list of APIs used to interface to the Silicon Labs RAIL library.

  • Flex Gecko 2.4 GHz, 20 dBm Range Test Demo Users Guide -- UG147

    Provides an easy way to evaluate the link budget of the Wireless Gecko EFR32 devices using Silicon Labs RAIL (RAIL) by performing a range test between two nodes using Range Test, a standalone test application. The range test demo implements Packet Error Rate (PER) measurement.

  • EFR32 Radio Configurator Guide -- AN971

    Describes the radio configurator GUI for RAIL framework applications. With it, you can create standard or custom radio configurations on which to run your RAIL-based applications. The role of each GUI item is explained.

  • Using RAIL Test

    Describes the functionality available in the RAILTest application.

  • EFR32 RF Evaluation Guide -- AN972

    Describes using RAILTest to evaluate radio functionality, as well as peripherals, deep sleep states, etc. With it you can fully evaluate the receiving and transmitting performance and test RF functionality of development kit hardware or custom hardware.

  • Porting RAIL Applications to RAIL Version 2.0 -- AN1113

    Describes the main drivers for RAIL 2.0 and key programmatic considerations when porting an application from RAIL 1.6 to RAIL 2.0.

  • Using RAIL for Wireless M-Bus Applications with EFR32 -- AN1119

    Describes using the Flex SDK for Wireless M-Bus development on EFR32 Wireless Geckos. Includes features and limitations as well as examples.

  • Power Amplifier Power Conversion Functions in RAIL 2.x -- AN1127

    Outlines how to account for the variation in output characteristics across custom boards and applications for the Silicon Labs EFR32 family of chips.

  • Using the Silicon Labs Dynamic Multiprotocol Demonstrations -- QSG155

    Shows how to demonstrate Dynamic Multiprotocol functionality using a Bluetooth LE smartphone app with either Zigbee-Bluetooth or RAIL-Bluetooth demo applications.
