1 RAIL Library {#mainpage}
2 ============
3 Silicon Labs Radio Abstraction Interface Layer (RAIL) is a library that can
4 be used as a generic interface for all Silicon Labs radio parts. By programming
5 against this API you are able to write code that easily ports across different
6 radio parts while having access to hardware acceleration wherever possible.
7 
8 # Introduction
9 
10 The RAIL library is mostly standalone with a few external dependencies.
11 Simplicity Studio application builder is not required but is recommended
12 because it simplifies building applications using
13 RAIL. Depending on the hardware platform
14 that you're using you may be required to
15 provide certain HAL functionality for the library to work properly. These
16 functions are described in hardware-specific documentation sections
17 and valid implementations can be found as a part of the provided
18 emlib/emdrv HAL layers. It is recommended that you use these
19 versions for maximum support, but you're free to re-implement them if necessary.
20 
21 # Using RAIL
22 
23 ## Features
24 
25 At a high level, the functionality supported by RAIL is shown below. For more
26 specifics on the APIs and how to use them see the module documentation.
27 
28 * \ref General
29  * Initialize the RAIL API layer.
30  * Collect entropy from the radio (if available).
31 * \ref Radio_Configuration
32  * Configure the radio frequency, packet format, channel configuration and
33  other PHY parameters.
34  * Query current PHY data rates and parameters like current channel.
35 * \ref State_Transitions
36  * Configure automatic radio state transitions and transition timings.
37 * \ref Auto_Ack
38  * Configure the radio for automatic acknowledgments.
39  * Load the auto ack payload.
40 * \ref System_Timing
41  * Get the current time in the RAIL timebase.
42  * Configure a timer to trigger an event callback after an absolute or
43  relative delay.
44  * Specify where within a packet its timestamp is desired.
45 * \ref Events
46  * Configure which radio or RAIL events the application wants to be told
47  about via callback.
48 * \ref Data_Management
49  * Allows the application to choose the type of data and the method of
50  data interaction through RAIL.
51 * \ref Receive
52  * Configure receive options like CRC checking.
53  * Start or schedule when to receive.
54  * Enable and configure \ref Address_Filtering for each packet.
55  * Enable and configure \ref Rx_Channel_Hopping for receiving packets
56  across several channels.
57 * \ref Transmit
58  * Configure the power amplifier (\ref PA) and set transmit power output.
59  * Load and send packet data, either immediately, scheduled, or using CSMA
60  or LBT.
61  * Control per-transmit options like CRC generation, ACK waiting, etc.
62 * \ref Antenna_Control
63  * Configuring multi-antenna selection.
64 * \ref Multiprotocol
65  * Manage time-sharing of the radio among different protocols.
66 * \ref Calibration
67  * APIs for handling various radio calibrations for optimal performance.
68 * \ref Rf_Sense
69  * Enable RF energy sensing of specified duration across the 2.4 GHz and/or
70  Sub-GHz bands (EFR32 only).
71 * \ref PTI
72  * Configure Packet Trace pins and serial protocol.
73  * Specify the stack protocol to aid network analyzer packet decoding.
74 * \ref Assertions
75  * Detecting and handling fatal runtime errors detected by RAIL.
76 * \ref Diagnostic
77  * Output debug signals like an unmodulated tone and a continuously modulated
78  stream of data.
79  * Configure crystal tuning for your radio.
80  * Fine-tune the radio tuner frequency.
81 
82 \ref Protocol_Specific hardware acceleration:
83 
84 * \ref IEEE802_15_4
85  * Configure the IEEE802.15.4 2.4GHz PHY.
86  * Configure node address and address filtering for IEEE 802.15.4.
87  * Configure auto ack for IEEE 802.15.4.
88 * \ref BLE
89  * Configure the Bluetooth Low Energy PHYs available for your device (1Mbps,
90  2Mbps, Coded PHY).
91  * Preamble, sync word and whitening adjustment function for connections.
92 * \ref Z_Wave
93  * Configure the Z-Wave PHY appropriate for your region.
94  * Configure your NodeId and HomeId for receive filtering and wakeup beam
95  filtering.

97 ## Getting Started Example
98 
99 Below is a simple example to show you how to initialize the RAIL library in your
100 application. Notice that this requires the channel configurations which can be
101 generated from the radio calculator in Simplicity Studio. By default, RAIL
102 interacts with data on a per-packet basis. For more information, see \ref
103 Data_Management.
104 
105 _Note: Before the radio can transmit, make sure the PA is initialized for your
106 platform, which is described in the Hardware-specific Configuration section
107 below._
108 
109 ~~~~~~~~~~~~~~~{.c}
110 #include "rail.h"
111 #include "rail_config.h" // Generated by radio calculator
112 
113 #define TX_FIFO_SIZE (128) // Any power of 2 from [64, 4096] on the EFR32
114 
115 static RAIL_Handle_t gRailHandle = NULL;
116 static RAIL_TxPower_t txPower = 200; // Default to 20 dBm
117 static uint8_t txFifo[TX_FIFO_SIZE];
118 
119 static const RAIL_TxPowerConfig_t railTxPowerConfig = { // May be const
120  // ... desired PA settings
121 };
122 
123 static void radioConfigChangedHandler(RAIL_Handle_t railHandle,
124  const RAIL_ChannelConfigEntry_t *entry)
125 {
126  bool isSubgig = (entry->baseFrequency < 1000000000UL);
127 
128  // ... handle radio configuration change, e.g., select the desired PA possibly
129  // using isSubgig to handle multiple configurations
130  RAIL_ConfigTxPower(railHandle, &railTxPowerConfig);
131 
132  // The TX power must be reapplied after changing the PA above.
133  RAIL_SetTxPowerDbm(railHandle, txPower);
134 }
135 
136 static void radioEventHandler(RAIL_Handle_t railHandle,
137  RAIL_Events_t events)
138 {
139  // ... handle RAIL events, e.g., receive and transmit completion
140 }
141 
142 #if MULTIPROTOCOL
143 static RAILSched_Config_t schedCfg; // Must never be const
144 static RAIL_Config_t railCfg = { // Must never be const
145  .eventsCallback = &radioEventHandler,
146  .protocol = NULL, // For BLE, pointer to a RAIL_BLE_State_t
147  .scheduler = &schedCfg, // For MultiProtocol, additional scheduler memory
148 };
149 #else
150 static RAIL_Config_t railCfg = { // Must never be const
151  .eventsCallback = &radioEventHandler,
152  .protocol = NULL,
153  .scheduler = NULL,
154 };
155 #endif
156 
157 // Initializes the radio out of startup so that it's ready to receive.
158 void radioInitialize(void)
159 {
160  // Initializes the RAIL library and any internal state it requires.
161  gRailHandle = RAIL_Init(&railCfg, NULL);
162 
163  // Configures calibration settings.
164  RAIL_ConfigCal(gRailHandle, RAIL_CAL_ALL);
165 
166  // Configures radio according to the generated radio settings.
167  RAIL_ConfigChannels(gRailHandle, channelConfigs[0], &radioConfigChangedHandler);
168 
169  // Configures the most useful callbacks and catches a few errors.
170  RAIL_ConfigEvents(gRailHandle,
171  RAIL_EVENTS_ALL,
172  RAIL_EVENT_TX_PACKET_SENT
173  | RAIL_EVENT_RX_PACKET_RECEIVED
174  | RAIL_EVENT_RX_FRAME_ERROR // invalid CRC
175  | RAIL_EVENT_RX_ADDRESS_FILTERED);
176 
177  // Sets automatic transitions to always receive once started.
178  RAIL_StateTransitions_t railStateTransitions = {
179  .success = RAIL_RF_STATE_RX,
180  .error = RAIL_RF_STATE_RX,
181  };
182  RAIL_SetRxTransitions(gRailHandle, &railStateTransitions);
183  RAIL_SetTxTransitions(gRailHandle, &railStateTransitions);
184 
185  // Sets up the transmit buffer.
186  RAIL_SetTxFifo(gRailHandle, txFifo, 0, TX_FIFO_SIZE);
187 }
188 ~~~~~~~~~~~~~~~
