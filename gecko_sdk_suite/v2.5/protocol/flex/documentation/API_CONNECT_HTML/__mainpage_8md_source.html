4 ## Silicon Labs Connect Stack
5 <P>Silicon Labs is developing products designed to meet the demands of customers as we move to an ever-connected world of devices in
6 the home, what is often referred to as the IoT (Internet of Things). At a high level the goals of IoT for Silicon Labs are to:</P>
7  <UL>
8 <LI>Connect all the devices in the home with best-in-class mesh networking, whether with Ember ZigBee PRO or other emerging standards.
9 <LI>Leverage the company's expertise in low-power, constrained devices.
10 <LI>Enhance established low-power, mixed-signal chips.
11 <LI>Provide low-cost bridging to existing Ethernet and Wi-Fi devices.
12 <LI>Enable cloud services and connectivity to smartphones and tablets that promote ease of use and a common user experience for
13 customers.
14 </UL>
15 <P>Achieving all of these goals will increase adoption rates and user acceptance for IoT devices in the Connected Home.
16 <P>One such challenge is managing devices requiring low power consumption, such as battery-powered devices where long battery life is
17 essential. To meet this challenge Silicon Labs has developed the Silicon Labs Connect stack. Connect provides a fully-featured, easily-customizable
18 wireless networking solution optimized for devices that require low power consumption and are used in a simple network
19 topology. Connect is configurable to be compliant with regional communications standards worldwide. Each RF configuration is designed
20 for maximum performance under each regional standard.
21 <P>The Silicon Labs Connect stack supports many combinations of radio modulation, frequency and data rates. The stack provides support
22 for end nodes, coordinators, and range extenders. It includes all wireless MAC (Medium Access Control) layer functions such as scanning
23 and joining, setting up a point-to-point or star network, and managing device types such as sleepy end devices, routers, and coordinators.
24 With all this functionality already implemented in the stack, users can focus on their end application development and not worry
25 about the lower-level radio and network details.
26 <P>The Connect stack should be used in applications with simple network topologies, such as a set of data readers feeding information
27 directly to a single central collection point (star or extended star topology), or a set of nodes in the same range exchanging data in a
28 single-hop fashion (direct devices or MAC devices). It does not provide a full mesh networking solution such as that provided by the
29 EmberZNet PRO or Silicon Labs Thread stacks.
30 <P>The Connect stack is part of the Silicon Labs Flex SDK (Software Development Kit), installed through Simplicity Studio. Connect runs
31 on top of RAIL (Radio Abstraction Interface Layer), also included with the Flex SDK. RAIL provides an intuitive, easily-customizable
32 radio interface layer that is designed to support proprietary or standards-based wireless protocols. For more information, see
33 <a href="https://www.silabs.com/documents/public/user-guides/ug103-13-app-dev-fundamentals-rail.pdf">UG103.13: RAIL Fundamentals</a>.
34 <P>The Connect stack supports efficient application development through its “building block” plug-in design. When used with the Simplicity
35 Studio IDE (Integrated Development Environment), developers can easily select the functions that should be included in the application.
36 The resulting applications are completely portable, in that they can be recompiled for different regions, different MCUs, and different
37 radios.</P>

39 ### Device Compatibility
40 
41 <P>The Connect stack supports the following device types:</P>
42 <UL>
43 <LI><B>Star coordinator</B>: The star coordinator forms and manages the star or extended star network. The star coordinator also communicates
44 with other range extenders and end nodes. Each Connect star network has a single coordinator.
45 <LI><B>Star range extender</B>: A device between the star coordinator and one or more star end nodes that can be used to extend the range of
46 the star end nodes. Each range extender can serve up to 32 star end nodes.
47 <LI><B>Star end node</B>: Joins to a star coordinator or a star range extender.
48 <LI><B>Direct device</B>: A device able to send and receive messages from other (direct) devices in range on the same PAN, with no star topology
49 restrictions. Such a device does not relay messages.
50 <LI><B>MAC device</B>: A device able to send and receive standard 802.15.4 messages from other 802.15.4 devices in range. Such a device
51 does not relay messages.
52 </UL>

54 ### Network Topology
55 
56 <P>The Connect stack supports three topologies, shown in the figure below:</P>
57 <UL>
58 <LI>Point to Point
59 <LI>Star
60 <LI>Extended Star
61 <LI>Single Hop Direct
62 <LI>Single Hop MAC
63 </UL>
64 <P>A Point to Point network provides simple communications between two devices: a star coordinator (blue node) and a star end node (white node).
65 <P>A Star network has a single star coordinator hub communicating with multiple star end nodes. All communication is through the star
66 coordinator.
67 <P>An Extended Star network includes a star range extender (red node) between the star coordinator and star end node(s) in one or more arms
68 of the star. Communications between the star coordinator and the far star end node(s) flow through one of the star range extenders.</P>
69 <P>A Single Hop Direct or MAC network includes two or more direct nodes (green node) communicating with each other in a single hop fashion.</P>

75 ### Stack Structure
76 
77 <P>The Connect stack provides code organized into three functional layers, as shown in the following figure:</P>
78 <UL>
79 <LI>PHY (physical)
80 <LI>MAC
81 <LI>Network
82 </UL>
83 <P>The PHY and MAC layers are based on the IEEE 802.15.4-2006 standard (<a href="http://standards.ieee.org/
84 findstds/standard/802.15.4-2006.html">IEEE 802.15.1-2006 Specification</a>). </P>
85 <P>The Network layer is based on a proprietary protocol.</P>

89 <P>Finally, the Connect Application Framework provides a complete tool and API infrastructure over the underlying stack layers. Functionality
90 within the Application Framework and the Connect stack layers is provided in the form of individual building blocks called plugins.
91 Details of the plugins for each layer are provided in the Silicon Labs Connect Application Framework API Reference included in the
92 stack documentation.</P>
